# /ace research — 시장/경쟁/고객 리서치

## 사용법

```
/ace research [이슈번호]
```

## 목적
사업 아이디어의 시장 환경, 경쟁 구도, 타겟 고객을 조사하여 구조화된 리서치 보고서를 작성한다.

## 선행 조건
- 태스크 생성 필수 (`/ace task start`)
- 아이디어 설명 (사용자 입력 또는 planning/ 폴더의 자료)

## 실행 흐름

### Step 1: 아이디어 수집

사용자로부터 아이디어를 수집한다:
- 해결하려는 문제 (Pain Point)
- 타겟 고객 (누구를 위한 것인가)
- 제안 솔루션 (어떻게 해결하는가)
- 차별점 (기존 대안 대비 왜 더 나은가)

### Step 2: 에이전트 호출

`researcher` 에이전트를 호출하여 다관점 리서치를 수행한다.

### Step 3: 산출물 작성

`workspace/tasks/{번호}/research.md`를 작성한다.

**리서치 관점:**

1. **시장 분석**
   - TAM/SAM/SOM 추정
   - 시장 트렌드 및 성장률
   - 규제/법률 환경

2. **경쟁 분석**
   - 직접/간접 경쟁자 목록
   - 각 경쟁자의 강점/약점
   - 가격 체계 비교
   - 포지셔닝 맵

3. **고객 분석**
   - 타겟 페르소나 (2-3개)
   - 고객의 현재 해결 방식 (대안)
   - 전환 비용 / 전환 동기
   - 지불 의향 (WTP) 추정

4. **기회와 위협**
   - SWOT 요약
   - 핵심 가정 (검증 필요 항목)
   - 진입 장벽

### Step 4: 상태 갱신

- research.md frontmatter: `status: done`
- taskDetail.json: `steps.research.status: completed`

## 도구 사용

- WebSearch: 시장 데이터, 경쟁사 정보, 트렌드
- WebFetch: 관련 보고서, 기사, 통계
- Read: 사용자 제공 자료
- Write: research.md 작성

## 완료 시 출력

```
═══ 리서치 완료 요약 ═══

[핵심 발견]
- 시장 규모, 주요 경쟁자, 타겟 고객 특성

[기회]
- 검증된 기회 포인트

[위험]
- 핵심 리스크/가정

[다음 단계]
- 비즈니스 모델링 시 반영할 사항
```
